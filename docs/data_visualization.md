# 데이터 시각화 규칙

현재 문서는 `dashboard.html`, `visualizations.js` 기준의 실제 구현 동작을 정리한 운영 문서다.  
문서에 적힌 규칙이 우선이 아니라, 현재 HTML/JS 구현을 기준으로 문서를 맞춘다.

## 0. 기본 개념

### 분석 대상 문항
- 결과를 시각화하는 중심 문항이다.
- 결과 카드 제목은 기본적으로 코드북의 `question_label`을 사용한다.
- 질문 본문이 있으면 `Q. {question_full}` 형식으로 함께 노출한다.

### 분석 기준 문항
- 분석 대상 문항의 결과를 그룹별로 나누어 비교할 때 사용하는 문항이다.
- 현재 대시보드는 단일응답 계열 그룹 문항과 시스템 컬럼 `survey_year`를 기준 문항 후보로 사용한다.
- 기준 문항을 선택하지 않으면 전체 결과만 보여준다.

### 지원 문항 유형
현재 결과 시각화를 지원하는 문항 유형은 아래와 같다.

- 객관식 단일
- 객관식 중복
- 객관식 순위
- 객관식 척도
- 주관식 숫자

아래 유형은 현재 결과 시각화 비지원으로 처리한다.

- 주관식 문자
- 기타 위 지원 목록에 포함되지 않는 유형

비지원 문항은 결과 영역에 "아직 지원하지 않는 문항 형식" 안내를 표시한다.

## 1. 공통 표시 규칙

### 비율과 반올림
- 비율은 내부 계산값을 유지하고, 표시 시점에만 소수점 첫째 자리까지 반올림한다.
- 반올림은 일반적인 `round half up` 방식으로 처리한다.
- 응답자 수 `N`은 반올림하지 않고 정수 그대로 표시한다.
- 같은 문항 안에서는 차트, 툴팁, 표가 동일한 비율 표시 규칙을 사용한다.

### 보기 정렬
- 기본 보기 순서는 코드북의 보기 정의 순서를 따른다.
- 코드북에 없는 예외 응답이 있으면 기존 보기 뒤에 추가한다.
- 전체 기준으로 응답 0건인 보기는 기본 차트와 표에서 숨긴다.
- 그룹 비교에서도 전체 기준 0건 보기는 함께 숨긴다.

### 공통 카드 구성
각 결과 카드는 문항 유형에 따라 아래 요소 중 일부 또는 전부를 사용한다.

- 문항 제목
- 질문 본문
- 차트
- 범례 또는 그룹 표시 컨트롤
- 데이터 테이블
- 보충 설명 문구

### 툴팁
- 툴팁은 브라우저 기본 툴팁이 아니라 커스텀 툴팁을 사용한다.
- 보기명, 비율, 응답 수, 그룹명 등 현재 차트 해석에 필요한 최소 정보를 보여준다.

### 그룹 표시 컨트롤
- 그룹 비교가 가능한 경우 체크박스형 범례를 사용한다.
- 개별 그룹 on/off가 가능하다.
- `전체 선택`, `전체 해제` 버튼을 함께 제공한다.
- 해제된 그룹은 색과 텍스트를 흐리게 표시해 비활성 상태를 구분한다.

### 기타 응답 보기
- 기타 응답용 텍스트 컬럼이 있으면 표의 해당 보기명 옆에 `응답 보기` 버튼을 노출할 수 있다.
- 버튼 클릭 시 기타 자유응답 목록을 모달로 보여준다.
- 모달에는 중복 제거된 텍스트 목록을 표시한다.

## 2. 유형별 시각화 규칙

### 2-1. 객관식 단일

#### 기본 시각화
- 가로 막대 차트를 사용한다.
- 각 보기는 `보기명 / 막대 / 비율` 구조로 표시한다.
- 막대 오른쪽에 비율 레이블을 함께 표시한다.

#### 그룹 비교
- 전체 결과는 회색 막대로 표시한다.
- 각 그룹 결과는 색상 선과 점으로 같은 행 위에 중첩 표시한다.
- 전체와 그룹을 동시에 비교하는 구조다.

#### 데이터 테이블
- 전체 보기에서는 `보기 / 비율 / 응답 수` 구조를 사용한다.
- 그룹 비교에서는 전체 열과 그룹별 열을 함께 보여준다.
- 표 하단에 합계 행을 둔다.

### 2-2. 객관식 중복

#### 집계 기준
- 중복 문항은 expanded 이진 컬럼을 기준으로 집계한다.
- 현재 구현은 선택값으로 `1`, `y`, `yes`, `true`, `선택`을 모두 선택 처리한다.
- 보기별 선택 여부를 기준으로 비율과 응답 수를 계산한다.

#### 기본 시각화
- 단일응답과 같은 가로 막대 차트 구조를 사용한다.
- 각 보기를 독립 항목으로 집계한다.

#### 그룹 비교
- 단일응답과 같은 전체 막대 + 그룹별 선/점 중첩 구조를 사용한다.

#### 데이터 테이블
- 전체 보기와 그룹 비교 모두 단일응답과 유사한 표 구조를 사용한다.
- 표 하단에 합계 행을 둔다.

#### 안내 문구
- 중복 문항 표 아래에 아래 취지의 안내 문구를 표시한다.
- 복수응답 문항이므로 보기별 비율 합은 100%를 초과할 수 있다.

### 2-3. 객관식 순위

#### 집계 기준
- 순위 문항은 순위별 expanded 컬럼을 기준으로 집계한다.
- 순위 expanded 컬럼명은 `__1순위` 표준 형식을 우선 사용한다.
- 현재 로더는 legacy 호환을 위해 `_1순위` 형식도 함께 인식한다.
- 응답자 수 `N`은 raw 값이 있거나 순위 expanded 값이 하나라도 있으면 포함한다.

#### 비순위 응답 처리
- raw에는 값이 있지만 순위별 expanded에 실제 순위값이 없는 응답은 비순위 응답으로 본다.
- 이런 응답은 전체 응답 수에는 포함한다.
- 다만 순위별 막대나 종합 순위 계산에는 포함하지 않는다.
- 필요 시 차트에서 `비순위 응답` 보조 구간으로 표시할 수 있다.

#### 기타 보기 처리
- `기타` 보기는 응답 수와 표에는 포함할 수 있다.
- 다만 종합 순위 계산에서는 제외한다.
- 기타 자유응답 텍스트가 있으면 `응답 보기` 버튼으로 모달을 열 수 있다.

#### 기본 시각화
- 보기별 가로 누적 막대 차트를 사용한다.
- 한 보기 안에서 `1순위`, `2순위`, `3순위` 등을 색상 구간으로 나눠 표시한다.
- 별도 순위 범례를 함께 보여준다.
- 상단에는 종합 순위를 요약한 텍스트 영역을 둘 수 있다.

#### 그룹 비교
- 그룹 비교 시에는 기본 누적 막대 대신 텍스트/표 기반 순위 비교 구조를 사용한다.
- 그룹별로 어떤 보기가 몇 위인지 요약해 비교한다.

#### 데이터 테이블
- 전체 보기에서는 순위별 `비율`, `응답 수`, `가중 점수`, `종합 순위`를 보여준다.
- 그룹 비교에서는 전체와 그룹별 순위 정보를 함께 보여준다.
- 합계 행의 `가중 점수`, `종합 순위`는 `-`로 표시한다.
- 표 하단에 가중 점수 산식 설명 문구를 둔다.

### 2-4. 객관식 척도

척도 문항은 `raw 척도`와 `derived 척도`를 구분해 처리한다.

#### 공통
- 기본 보기 모드는 `분포 보기`다.
- `평균 보기`로 전환할 수 있다.
- 홀수 척도이면서 derived가 아닌 경우 `중간값 제외 보기`를 제공할 수 있다.
- 중간값 제외 보기는 분포 표시만 바꾸며, 원본 통계값 자체를 바꾸지는 않는다.

#### raw 척도
- 응답 분포를 100% 누적 막대로 표시한다.
- 부정/긍정 방향이 있는 척도는 diverging 색상 팔레트를 사용한다.
- 분포 보기에서는 점수별 구간 비율과 함께 부정/긍정 요약 정보를 보여준다.
- 평균 보기에서는 축 위에 평균 마커를 표시한다.

#### raw 척도 그룹 비교
- 그룹 비교 컨트롤은 별도 범례 대신 카드 상단 컨트롤 영역에 배치된다.
- 그룹 비교 시 분포 또는 평균을 그룹별로 비교한다.

#### derived 척도
- `type = scale` 이면서 `role = derived`인 경우 derived 척도로 본다.
- 연속형 계산값을 점수 범위 안에서 시각화한다.
- 현재 구현은 바이올린 차트 스타일을 사용한다.
- `Q1`, `Q2(중앙값)`, `Q3`와 평균 마커를 함께 표시할 수 있다.

#### 척도형 여러 문항 비교
- 같은 척도 길이의 문항끼리는 `여러 문항 한 번에 비교하기`를 사용할 수 있다.
- 비교 가능한 문항만 버튼이 활성화된다.
- 비교 문항 선택은 전용 모달에서 수행한다.
- 그룹이 없고 분포 보기 상태이면 문항 간 분포 비교를 보여줄 수 있다.
- 그룹이 있으면 여러 문항 비교는 평균 비교 중심으로 동작한다.
- 그룹이 있는 여러 문항 비교에서는 분포 보기를 비활성화하고 평균 보기로 고정한다.

#### 데이터 테이블
- 단일 척도 표는 평균, 점수별 비율/응답 수, 응답자 수를 보여준다.
- 여러 문항 비교 표는 문항별 평균과 분포 또는 그룹별 평균 정보를 보여준다.

### 2-5. 주관식 숫자

#### 기본 시각화
- 그룹이 없을 때는 `히스토그램`, `박스플롯` 두 가지 보기 모드를 지원한다.
- 기본 모드는 히스토그램이다.

#### 히스토그램
- 사용자가 구간 시작값과 구간 간격을 조정할 수 있다.
- 막대 아래에는 구간 기준 설명을 함께 제공한다.
- 평균 마커와 사분위 기준선을 함께 표시할 수 있다.

#### 박스플롯
- 최소값~최대값 whisker, Q1~Q3 box, 중앙값 선, 평균 마커를 표시한다.
- 각 요소 의미를 설명하는 안내 문구를 함께 제공한다.

#### 그룹 비교
- 그룹이 있으면 박스플롯만 사용한다.
- 전체 행과 그룹별 행을 같은 축 위에서 함께 비교한다.

#### 데이터 테이블
- 평균, 최소값, Q1, 중앙값, Q3, 최대값, 응답자 수를 제공한다.

## 3. 상호작용 규칙

### 그룹 on/off
- 그룹 비교 차트에서 표시할 그룹을 사용자가 직접 선택할 수 있다.
- 선택 해제된 그룹은 차트, 범례, 일부 표 강조에서 제외된다.

### 척도 보기 전환
- 척도 문항은 `분포 보기`, `평균 보기`를 전환할 수 있다.
- 일부 raw 척도는 `중간값 제외 보기`를 함께 제공한다.

### 숫자형 보기 전환
- 숫자형 문항은 `히스토그램`, `박스플롯` 전환을 지원한다.
- 그룹 비교 시에는 박스플롯으로 고정한다.

### 여러 문항 비교
- 척도형 문항은 대상 문항 여러 개를 한 번에 비교할 수 있다.
- 비교 문항은 현재 선택 문항과 동일한 척도 길이 조건을 만족해야 한다.

## 4. 기타 응답 처리 규칙

### 기타 텍스트 컬럼 탐색
- 기타 자유응답 텍스트는 전용 expanded 컬럼에서 읽는다.
- 현재 구현은 아래와 같은 이름 패턴을 순차적으로 허용한다.
  - `__기타_텍스트`
  - `__기타 텍스트`
  - `_기타_텍스트`
  - `_텍스트`
- 위 패턴에 정확히 맞지 않아도, 문항명과 기타 텍스트 의미가 함께 드러나는 경우 느슨한 fallback 탐색을 수행한다.

### 기타 응답 표시
- 기타 자유응답이 존재하면 표에서 `응답 보기` 버튼을 노출할 수 있다.
- 모달에서는 중복 제거된 텍스트 목록을 보여준다.
- 기타 응답은 차트 표시보다 상세 확인 보조 기능에 가깝게 다룬다.

## 5. 예외 및 비지원 처리

### 데이터 부족
- 문항은 지원 유형이더라도 실제 집계에 필요한 컬럼이 없으면 결과를 그리지 못할 수 있다.
- 이 경우 현재 구현은 비지원/표시 불가 상태의 안내 영역을 보여준다.

### 순위 문항 컬럼 불완전
- 순위형은 순위별 expanded 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.
- raw 값만 있다고 해서 순위 차트를 강제로 생성하지 않는다.

### 중복 문항 컬럼 불완전
- 중복형은 보기별 expanded 이진 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.

## 6. 문서 운영 원칙

- 문서보다 실제 구현이 우선이다.
- `dashboard.html`, `visualizations.js` 동작이 바뀌면 이 문서도 함께 갱신한다.
- 문서에는 기획안이 아니라 "현재 화면에서 실제로 보이는 규칙"을 적는다.
