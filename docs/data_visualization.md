# 데이터 시각화 디자인 시스템

> **범위**: 시각화 영역의 디자인 토큰, 컴포넌트 스펙, 렌더링 규칙, 데이터 집계·계산·예외 처리 규칙을 함께 다룬다.
> **연계**: 공통 디자인 토큰(폰트, 중립 색상, 반경, 그림자)은 `design_system.md`를 참조한다. 이 문서에는 시각화 영역 전용 적용 기준만 기술한다.
> **AI 작업 규칙**: 이 문서에 정의된 토큰과 컴포넌트만 사용한다. `design_system.md`와 이 문서에 없는 색상·크기·간격을 임의로 추가하지 않는다. 새 토큰이 필요하면 반드시 사용자에게 확인을 요청한다.

---

## 0. 기본 개념

### 분석 대상 문항
- 결과를 직접 시각화해서 보여주는 중심 문항이다.
- 결과 카드 제목은 기본적으로 코드북의 `question_label`을 사용한다.
- 질문 원문이 있으면 `Q. {question_full}` 형식으로 함께 표시한다.

### 분석 기준 문항
- 분석 대상 문항의 결과를 그룹별로 나누어 비교할 때 사용하는 문항이다.
- 현재 대시보드는 단일응답 계열 그룹 문항과 시스템 컬럼 `survey_year`를 기준 문항 후보로 사용한다.
- 기준 문항을 선택하지 않으면 전체 결과만 보여준다.

### 지원 문항 유형
- 객관식 단일
- 객관식 중복
- 객관식 순위
- 객관식 척도
- 주관식 숫자
- 주관식 비율 배분

### 비지원 문항 유형
- 주관식 문자
- 기타 위 지원 목록에 포함되지 않는 유형
- 비지원 문항은 결과 영역에 `아직 지원하지 않는 문항 형식` 안내를 표시한다.

### 1-2. 컬러 팔레트 용도

#### 단일 색상 그래프
- 단일 색상을 사용하는 경우(가로/세로 막대 기본 차트, 히스토그램 등) `neutral-700`을 기본으로 한다.

#### 여러 색상이 필요한 그래프
- **그룹 비교**: 그룹 순서대로 `color-1` → `color-2` → … → `color-20` 순환 할당. 최대 20개 그룹까지 색상 구분 가능.
- **객관식 단일 — 누적 막대 / 원형 그래프**: 보기 순서대로 `color-1` → `color-2` → … 순환 할당해 보기별 색상을 구분.

#### 순위 계열
- 누적 막대 차트에서 1/2/3순위 구간을 색상으로 구분할 때 사용한다. neutral 계열만 사용하되 명도로 순위를 구분한다.

| 순위 | 토큰 |
|------|------|
| 1순위 | `neutral-700` |
| 2순위 | `neutral-400` |
| 3순위 | `neutral-300` |
| 4순위 이하 | `neutral-200` |

#### 척도 계열
- 1점(최저)은 `color-2`(붉은 계열), 중간점은 `neutral-300`, 최고점은 `color-1`(푸른 계열)을 기준으로 하고, 그 사이는 자연스러운 그라데이션 값으로 채운다.
- 5점 척도와 7점 척도는 아래 가이드를 따르고, 그 외 척도는 같은 방식으로 적당히 지정한다.

**5점 척도**

| 점수 | 토큰 |
|------|------|
| 1 | `color-2` |
| 2 | `low-3` |
| 3 | `neutral-300` |
| 4 | `high-3` |
| 5 | `color-1` |

**7점 척도**

| 점수 | 토큰 |
|------|------|
| 1 | `color-2` |
| 2 | `low-4` |
| 3 | `low-2` |
| 4 | `neutral-300` |
| 5 | `high-2` |
| 6 | `high-4` |
| 7 | `color-1` |

**그 외 척도**: 위 기준점(`color-2` → `neutral-300` → `color-1`)을 유지하면서 단계 수에 맞게 중간 토큰을 적절히 배정한다.

#### 박스플롯 계열
- 박스 내부 구간을 두 가지 색상으로 나눠 분포의 방향성을 표현한다.

| 구간 | 토큰 |
|------|------|
| Q1–Q2 (하위 구간) | `color-2` (붉은 계열) |
| Q2–Q3 (상위 구간) | `color-1` (푸른 계열) |

#### 척도 평균 그래프 계열
- 분포 막대 배경의 각 구간 색상은 neutral 계열만 사용하며, 양 끝(최저·최고 점수)은 `neutral-400`, 중앙은 `neutral-200`, 그 사이는 자연스러운 그라데이션으로 채운다.
- 단, 5점 척도는 중앙(3점)을 `neutral-300`으로 한다.

#### 차트

##### 공통

> **방향 정의**: 막대가 가로로 뻗는 차트를 **가로 막대**, 막대가 세로로 뻗는 차트를 **세로 막대**라 한다. 가로 막대에서 보기명은 왼쪽, 비율은 오른쪽으로 늘어난다. 세로 막대에서 보기명은 하단, 비율은 위로 올라간다.

- 막대 border-radius: `--radius-8` (8px). 누적 막대는 첫 번째·마지막 구간에만 양끝 적용.
- 수치 레이블: `Label-1-strong` (→ 1-1 타이포그래피 참조). 차트 요소가 너무 좁을 때는 표시하지 않음.
- 보기명: `Label-1` (→ 1-1 타이포그래피 참조), `neutral-900`
- 막대 그래프 (1~5번, 11번 히스토그램) 공통 축 스타일: 축 색상 `neutral-300` / 눈금 색상 `neutral-200`
- 막대 hover: opacity 0.85
- 막대 전환 애니메이션: `width 0.2s ease` (가로 막대) / `height 0.2s ease` (세로 막대)

## 3. 공통 표시 규칙

### 비율과 반올림
- 비율은 내부 계산값을 유지하고, 표시 시점에만 소수점 첫째 자리까지 반올림한다.
- 반올림은 일반적인 `round half up` 방식으로 처리한다.
- 응답자 수 `N`은 반올림하지 않고 정수 그대로 표시한다.
- 같은 문항 안에서는 차트, 툴팁, 표가 동일한 비율 표시 규칙을 사용한다.
- 툴팁에서는 응답 비율을 `xx.x%`, 응답 수를 `N=xxx` 형식으로 표시한다.

### 보기 정렬
- 기본 보기 순서는 코드북의 보기 정의 순서를 따른다.
- 코드북에 없는 예외 응답이 있으면 기존 보기 뒤에 추가한다.
- 전체 기준으로 응답 0건인 보기는 기본 차트와 표에서 숨긴다.
- 그룹 비교에서도 전체 기준 0건 보기는 함께 숨긴다.

### 공통 카드 구성
- 문항 제목
- 질문 원문
- 컨트롤바
- 차트
- 범례 또는 그룹 표시 컨트롤
- 데이터 테이블 (숨기기/펼치기 가능)
- 보충 설명 문구

### 공통 그룹 범례 규칙
- 그룹 비교가 가능한 경우 체크박스형 그룹 범례 또는 그룹 컨트롤을 사용한다.
- 그룹 비교 시 그룹명은 `분석 기준 문항명: 그룹 값` 형식으로 표기한다.
  - 예: `성별: 여`, `세대: Z세대`
- 개별 그룹 on/off가 가능하다.
- `전체 선택`, `전체 해제` 버튼을 함께 제공한다.
- 해제된 그룹은 범례에서 색과 텍스트를 흐리게 표시해 비활성 상태로 구분하고, 차트와 표에서는 숨긴다.

### 기타 응답 보기
- 기타 응답용 텍스트 컬럼이 있으면 데이터 테이블의 해당 보기명 옆에 `응답 보기` 버튼을 표시할 수 있다.
- 버튼을 누르면 기타 자유응답 목록을 모달로 보여준다.
- 모달에는 중복 제거 없이 실제 수집된 텍스트 목록을 그대로 보여준다.

## 4. 유형별 시각화 규칙

### 4-1. 객관식 단일

#### 집계 기준
- 각 보기별 선택 응답 수를 집계하고, 전체 응답 수 대비 비율을 계산한다.

#### 차트
- 컨트롤바는 차트 위에 위치하며, 그래프 유형 드롭다운과 응답 비율 내림차순 정렬 옵션을 둔다.
- 기본 차트는 가로 막대 차트를 사용한다.
- 세로축 역할 위치에는 보기명이 놓인다.
- 가로축 역할 위치에는 `0%~100%` 기준의 비율 크기가 반영된다.
- 각 보기는 `보기명 / 막대` 구조로 표시하고, 비율 데이터 레이블은 항상 막대 끝 바깥쪽 오른쪽에 둔다.
- 비율 데이터 레이블은 강조 정보로 보고 `Label-1-strong`(14px, Semibold 600)을 사용한다.
- 보기 개수가 6개 이하인 문항에 한해 그래프 유형 선택이 활성화된다. 선택지: `가로 막대 / 세로 막대 / 원형 그래프 / 누적 막대그래프`
- 보기 개수가 7개 이상이면 그래프 유형 선택은 비활성화되고 가로 막대로 고정된다.

#### 툴팁
- 기본 툴팁은 `보기명`, `비율`, `응답 수`를 보여준다.
- 그룹 비교 툴팁은 `그룹명`, `보기명`, `비율`, `응답 수`를 보여준다.

#### 범례 및 그룹 비교
- 전체 결과는 회색 막대로 표시한다.
- 그룹 비교 시에는 각 그룹 결과를 색상 꺾은선과 점으로 같은 행 위에 중첩 표시한다.
- 전체와 그룹을 동시에 비교하는 구조다.
- 그룹 비교 시에는 공통 그룹 범례 규칙을 따른다.

#### 데이터 테이블
- 전체 보기에서는 `보기`, `응답 비율(%)`, `응답 수(명)` 구조를 사용한다.
- 그룹 비교에서는 전체 열과 그룹별 열을 함께 보여준다.
- 표 하단에 합계 행을 둔다.
- 기타 자유응답 텍스트가 있으면 해당 보기명 옆에 `응답 보기` 버튼을 표시한다.

### 4-2. 객관식 중복

#### 집계 기준
- 중복 문항은 expanded 이진 컬럼을 기준으로 집계한다.
- 현재 구현은 선택값으로 `1`, `y`, `yes`, `true`, `선택`을 모두 선택 처리한다.
- 보기별 선택 여부를 기준으로 비율과 응답 수를 계산한다.

#### 차트
- 컨트롤바에서 보기 순서를 응답 비율 내림차순으로 정렬할 수 있다. 기본은 코드북 순서다.
- 객관식 단일과 같은 가로 막대 차트 구조를 사용한다.
- 세로축 역할 위치에는 보기명이 놓인다.
- 가로축 역할 위치에는 `0%~100%` 기준의 비율 크기가 반영된다.
- 각 보기를 독립 항목으로 집계한다.
- 비율 데이터 레이블은 객관식 단일과 같은 기준을 따른다. 항상 막대 끝 바깥쪽 오른쪽에 둔다.
- 비율 데이터 레이블은 `Label-1-strong`(14px, Semibold 600)을 사용한다.
- 보기 개수가 6개 이하인 문항에 한해 세로 막대 차트로 전환할 수 있다.

#### 툴팁
- 기본 툴팁은 `보기명`, `비율`, `응답 수`를 보여준다.
- 그룹 비교 툴팁은 `그룹명`, `보기명`, `비율`, `응답 수`를 보여준다.

#### 범례 및 그룹 비교
- 객관식 단일과 같은 전체 막대 + 그룹별 꺾은선/점 중첩 구조를 사용한다.
- 그룹 비교 시에는 공통 그룹 범례 규칙을 따른다.

#### 데이터 테이블
- 전체 보기에서는 `보기`, `응답 비율(%)`, `응답 수(명)` 구조를 사용한다.
- 그룹 비교에서는 응답자 전체와 각 그룹의 `응답 비율(%)`, `응답 수(명)` 정보를 함께 보여준다.
- 표 하단에 합계 행을 둔다.
- 기타 자유응답 텍스트가 있으면 해당 보기명 옆에 `응답 보기` 버튼을 표시한다.

#### 안내 문구
- `객관식 중복 응답 문항으로, 보기별 비율 합계는 100%를 초과할 수 있습니다.`

### 4-3. 객관식 순위

#### 집계 기준
- 순위 문항은 순위별 expanded 컬럼을 기준으로 집계한다.
- 순위 expanded 컬럼명은 `__1순위` 표준 형식을 우선 사용한다.
- 현재 로더는 legacy 호환을 위해 `_1순위` 형식도 함께 인식한다.
- 응답자 수 `N`은 raw 값이 있거나 순위 expanded 값이 하나라도 있으면 포함한다.
- 순위별 비율 계산의 모수는 해당 순위를 실제로 선택한 응답자 수가 아니라, 해당 문항에 참여한 전체 응답자 수를 사용한다.

#### 가중 점수 계산
- 가중치는 최하위 순위부터 1점씩 시작해 순위가 올라갈수록 2씩 증가한다.
  - 예: 3순위까지인 경우 1순위=5점, 2순위=3점, 3순위=1점
  - 예: 4순위까지인 경우 1순위=7점, 2순위=5점, 3순위=3점, 4순위=1점
- 보기별 가중 평균 점수 = 각 순위 응답 수 × 해당 순위 가중치의 합 ÷ 전체 응답자 수
- 종합 순위는 가중 평균 점수 기준 내림차순으로 결정하며, 동률이면 공동 순위로 표시한다.

#### 비순위 응답 처리
- raw에는 값이 있지만 순위별 expanded에 실제 순위값이 없는 응답은 비순위 응답으로 본다.
- 이런 응답은 전체 응답 수에는 포함한다.
- 다만 가중 평균 점수와 종합 순위 계산에는 포함하지 않는다.

#### 기타 보기 처리
- `기타` 보기는 응답 수와 표에는 포함할 수 있다.
- 다만 종합 순위 계산에서는 제외한다.

#### 차트
- 컨트롤바에서 보기 순서를 응답 비율(가중 평균 점수) 내림차순으로 정렬할 수 있다. 기본은 코드북 순서다.
- 기본 차트는 보기별 가중 평균 점수를 나타내는 가로 롤리팝 차트를 사용한다.
- 세로축 역할 위치에는 보기명이 놓인다.
- 가로축 역할 위치에는 가중 평균 점수 크기가 반영된다.
- 각 보기의 우측 레이블에는 `가중 평균 점수(종합 순위)`를 표기한다. 예: `3.2(1위)`
- 차트 하단에는 가중 평균 산출 방식을 간단히 표기한다. 예: `가중 평균 = (1순위×5 + 2순위×3 + 3순위×1) ÷ 전체 응답자 수`
- 컨트롤바에서 롤리팝과 누적 막대 차트를 토글할 수 있으며, 각각 가로/세로 방향을 선택할 수 있다.
- 누적 막대 차트에서는 각 보기 안에서 `1순위`, `2순위`, `3순위` 등을 색상 구간으로 나눠 표시한다.
- 가로 누적 막대 차트의 전체 비율 수치는 가로 막대형 그래프 공통 수치 레이블 규칙을 따른다.

#### 툴팁
- 롤리팝 차트 툴팁은 `보기명`, `가중 평균 점수`, `종합 순위`를 보여준다.
- 누적 막대 차트 순위 구간 툴팁은 `보기명`, `해당 순위`, `비율`, `응답 수`를 보여준다.

#### 범례 및 그룹 비교
- 누적 막대 차트 보기에서는 순위별 색상 구간을 설명하는 순위 범례를 표시한다.
- 순위 범례는 체크 해제가 가능하며, on/off는 차트와 표에 반영한다.
- `1순위 / 2순위 / 3순위` 체크박스와 `전체 선택 / 전체 해제`를 함께 제공한다.
- 그룹 비교 시에는 각 그룹의 1순위 응답만 비교 대상으로 사용한다.
- 그룹 비교 차트는 보기별 1순위 선택 비율을 기준으로 그려, 그룹별로 가장 선호되는 항목을 바로 확인할 수 있게 한다.
- 그룹 비교 차트의 보기 순서는 응답자 전체의 1순위 선택 비율이 높은 순서로 정렬한다.
- 그룹 비교 상태에서는 2순위 이하 응답을 차트 비교 기준에 섞지 않는다.
- 그룹 비교 시에는 공통 그룹 범례 규칙을 따른다.

#### 데이터 테이블
- 보기별로 순위별 `응답 비율(%)`, `응답 수(명)`을 먼저 나열하고, 가장 오른쪽에 `가중 평균`, `종합 순위` 컬럼을 둔다.
- 그룹 비교에서는 응답자 전체의 순위 정보와 그룹별 순위 정보를 함께 보여준다.
- 표 하단에 합계 행을 둔다.
- 합계 행의 `가중 평균`, `종합 순위`는 `-`로 표시한다.
- 기타 자유응답 텍스트가 있으면 해당 보기명 옆에 `응답 보기` 버튼을 표시한다.

### 4-4. 객관식 척도

척도 문항은 `raw 척도`와 `derived 척도`를 구분해 처리한다.

#### 공통
- 기본 보기 모드는 `분포 보기`다.
- `평균 보기`로 전환할 수 있다.
- 홀수 척도이면서 derived가 아닌 경우 `중간값 제외 보기`를 제공할 수 있다.
- 중간값 제외 보기는 분포 표시만 바꾸며, 원본 통계값 자체를 바꾸지는 않는다.

#### raw 척도 집계 기준
- 각 점수별 응답 수를 집계하고, 전체 응답 수 대비 비율을 계산한다.
- 동시에 평균값도 계산한다.

#### raw 척도 차트
- `분포 보기`와 `평균 보기` 두 가지 모드를 사용한다.
- 분포 보기는 점수별 응답 분포를 100% 누적 막대로 표시한다.
- 점수 구간은 척도형 팔레트를 사용한다. 색상 배정은 **5. 유형별 컬러 매핑**을 따른다.
- 분포 보기에서는 점수별 비율과 함께 하위 척도 응답 합계, 상위 척도 응답 합계 요약 정보를 보여준다.
- 평균 보기에서는 척도 개수만큼 눈금이 있는 축을 표시하고, 그 위에 평균 마커를 둔다.

#### raw 척도 툴팁
- 분포 보기에서는 `점수값 또는 점수 레이블`, `비율`, `응답 수`를 보여준다.
- 평균 보기에서는 `평균값`, `응답 수`를 보여준다.
- 분포 보기 그룹 비교 툴팁은 `그룹명`, `점수값 또는 점수 레이블`, `비율`, `응답 수`를 보여준다.
- 평균 보기 그룹 비교 툴팁은 `그룹명`, `평균값`, `응답 수`를 보여준다.

#### raw 척도 범례 및 그룹 비교
- 그룹 비교 컨트롤은 별도 범례 대신 카드 상단 컨트롤 영역에 배치한다.
- 상단 컨트롤 영역에서 체크박스로 그룹 표시를 해제할 수 있다.
- 그룹 비교 시에는 같은 형식의 차트를 그룹별로 세로로 나열해 비교한다.

#### raw 척도 데이터 테이블
- 단일 문항 전체 보기에서는 `점수`, `응답 비율(%)`, `응답 수(명)`, `평균` 컬럼을 사용한다.
- `평균` 컬럼은 헤더에는 나란히 두되, 값은 합계 행에만 표시하고 점수별 행의 `평균` 칸은 비워 둔다.
- 단일 문항 전체 보기에는 합계 행이 있다.
- 단일 문항 그룹 비교에서는 응답자 전체와 각 그룹의 `점수`, `응답 비율(%)`, `응답 수(명)`, `평균` 컬럼을 함께 보여준다.
- 그룹 비교에서도 `평균`은 응답자 전체와 각 그룹 블록의 합계 행에만 값을 표시한다.
- 단일 문항 그룹 비교에도 응답자 전체와 각 그룹 블록별 합계 행이 있다.
- 여러 문항 비교이면서 그룹 비교가 없을 때는 `문항명`, 각 점수별 `응답 비율(%)`, `응답 수(명)`, `평균`, `전체 응답 수(명)`를 보여준다.
- 여러 문항 비교이면서 그룹 비교가 있을 때는 `문항명`과 응답자 전체 및 각 그룹의 `평균`, `응답 수(명)`를 보여준다.

#### derived 척도 집계 기준
- 계산된 연속형 점수를 기준으로 최소값, 최대값, 평균, 사분위수(`Q1`, `Q2`, `Q3`)를 계산한다.

#### derived 척도 차트
- 바이올린 차트 스타일을 사용한다.
- `type = scale` 이면서 `role = derived`인 경우 derived 척도로 본다.
- 연속형 계산값을 점수 범위 안에서 시각화한다.
- `Q1`, `Q2`, `Q3`를 표시한다.
- 평균 보기에서는 평균 마커를 함께 표시한다.

#### derived 척도 툴팁
- 바이올린 영역 자체에는 별도 툴팁을 두지 않는다.
- 사분위수 마커에는 `사분위명`, `사분위값`을 보여준다.
- 사분위수 마커 표시는 `Q1`, `Q2`, `Q3`처럼 간단히 표시하되, 툴팁에서는 `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`처럼 친절한 이름을 사용한다.
- 평균 보기에서는 `평균값`, `응답 수`를 보여준다.
- 그룹 비교 평균 보기 툴팁은 `그룹명`, `평균값`, `응답 수`를 보여준다.
- 그룹 비교 사분위수 마커 툴팁은 `그룹명`, `사분위명`, `사분위값`을 보여준다.

#### derived 척도 범례 및 그룹 비교
- derived 척도도 그룹 비교를 지원한다.
- 전체 분포와 그룹별 분포를 같은 점수 축 기준으로 그룹별 차트를 세로로 나열해 비교한다.
- 각 그룹은 같은 형식의 바이올린 분포와 사분위수 지표를 표시한다.
- 평균 보기에서는 전체와 각 그룹의 평균 마커를 함께 비교할 수 있다.
- 그룹 비교 시에는 상단 그룹 컨트롤 영역에서 표시 그룹을 조정한다.

#### derived 척도 데이터 테이블
- 단일 문항일 때는 `문항명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)`을 보여준다.
- 단일 문항 그룹 비교일 때는 `그룹명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)`을 보여주고, 표 상단에 응답자 전체 행을 별도로 둔다.
- 여러 문항 비교이면서 그룹 비교가 없을 때는 `문항명`, 각 점수별 `응답 비율(%)`, `응답 수(명)`, `평균`, `전체 응답 수(명)`를 보여준다.
- 여러 문항 비교이면서 그룹 비교가 있을 때는 `문항명`과 응답자 전체 및 각 그룹의 `평균`, `응답 수(명)`를 보여준다.

#### 척도형 여러 문항 비교
- 같은 척도 길이의 문항끼리는 `여러 문항 한 번에 비교하기`를 사용할 수 있다.
- 문항 선택 영역에 올린 문항들이 모두 비교 가능한 같은 척도형이면 버튼이 자동으로 활성화된다.
- 비교할 수 없는 문항이 하나라도 섞이면 버튼은 비활성화된다.
- 그룹이 없고 분포 보기 상태이면 문항 간 분포 비교를 보여준다.
- 그룹이 있으면 여러 문항 비교는 평균 비교 중심으로 동작한다.
- 그룹이 있는 여러 문항 비교에서는 분포 보기를 비활성화하고 평균 보기로 고정한다.

### 4-5. 주관식 숫자

#### 집계 기준
- 숫자 응답값을 기준으로 최소값, 최대값, 평균, 사분위수(`Q1`, `Q2`, `Q3`)를 계산한다.
- 히스토그램 보기에서는 구간별 응답 수와 비율을 계산한다.

#### 차트
- 그룹이 없을 때는 `히스토그램`, `박스플롯` 두 가지 보기 모드를 지원한다.
- 기본 모드는 히스토그램이다.

#### 히스토그램
- 사용자가 구간 시작값과 구간 간격을 조정할 수 있다.
- `구간 시작값과 간격을 조정해 분포를 원하는 기준으로 확인할 수 있습니다.` 안내 문구를 표시한다.
- 가로축은 각 구간의 범위를 나타낸다.
- 세로축은 각 구간의 응답 수 크기를 나타낸다.
- 평균 마커와 사분위수 마커를 함께 표시한다.

#### 박스플롯
- 가로축은 값의 범위를 읽는 기준 역할을 한다.
- 최소값~최대값 whisker, `Q1~Q3` box, 중앙값 기준선, 평균 마커를 표시한다.
- 평균 마커는 중앙값 기준선보다 위에 배치해 가려지지 않도록 한다.
- 각 요소 의미를 설명하는 안내 문구를 함께 제공한다.

#### 툴팁
- 히스토그램에서는 `구간 범위`, `응답 수`를 보여준다.
- 평균 마커 툴팁에는 `평균값`, `응답 수`를 보여준다.
- 사분위수 마커 툴팁에는 `사분위명`, `사분위값`을 보여준다.
- 사분위수 마커 표시는 `Q1`, `Q2`, `Q3`처럼 간단히 표시하되, 툴팁에서는 `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`처럼 친절한 이름을 사용한다.
- 박스플롯에서는 `최소값`, `Q1(하위 25%)`, `Q2(중앙값)`, `Q3(상위 25%)`, `최대값`, `평균값`처럼 값의 이름과 값을 함께 보여준다.

#### 범례 및 그룹 비교
- 그룹이 있으면 박스플롯만 사용한다.
- 전체 결과와 그룹별 결과를 같은 축 위에서 함께 비교한다.
- 그룹 비교 시에는 공통 그룹 범례 규칙을 따른다.

#### 데이터 테이블
- 기본 보기에서는 `문항명`, `최소값`, `Q1`, `Q2(중앙값)`, `Q3`, `최대값`, `평균`, `응답 수(명)` 순서로 제공한다.
- 그룹 비교에서는 `문항명` 대신 `그룹명` 컬럼을 사용하고, 표 상단에 응답자 전체 행을 별도로 둔다.

### 4-6. 주관식 비율 배분

#### 집계 기준
- raw 컬럼에는 배분 항목 순서대로 `40|60`처럼 파이프 구분 숫자값이 들어갈 수 있다.
- expanded 컬럼이 있으면 `{원문항 라벨}__{배분 항목명}` 컬럼의 항목별 숫자값을 우선 사용한다.
- expanded 컬럼이 없으면 raw 값의 파이프 순서를 코드북의 `response_options` 순서에 맞춰 해석한다.
- 각 응답자의 항목별 합계가 100인 응답만 유효 응답으로 집계한다.
- 항목별 평균 배분값은 유효 응답자별 항목 값을 평균해 계산한다.

#### 차트
- 객관식 척도 단일 문항과 유사한 가로 누적 막대 그래프를 사용한다.
- 막대 전체는 100% 기준이며, 각 구간의 너비는 항목별 평균 배분값을 나타낸다.
- 각 막대 하단에는 항목별 레이블과 평균 배분값을 표시한다.
- 그룹 비교 시에는 그룹별 가로 누적 막대를 세로로 나열한다.

#### 툴팁
- 항목 구간과 하단 레이블 툴팁에는 `그룹명`, `배분 항목`, `평균 배분값`, `응답 수`를 보여준다.

#### 범례 및 그룹 비교
- 항목 구간은 배분 항목별 색상으로 구분한다.
- 그룹 비교 시에는 공통 그룹 범례 규칙을 따른다.

#### 데이터 테이블
- 기본 보기에서는 `배분 항목`, `평균 배분값`, `응답 수(명)`을 보여준다.
- 그룹 비교에서는 응답자 전체와 각 그룹의 `평균 배분값`, `응답 수(명)`을 함께 보여준다.

## 6. 상호작용 규칙

### 그룹 on/off
- 그룹 비교 차트에서 표시할 그룹을 사용자가 직접 선택할 수 있다.
- 선택 해제된 그룹은 범례에서는 흐리게 표시되고, 차트와 표에서는 숨겨진다.

### 그룹 설정
- `그룹 설정` 버튼을 클릭하면 모달을 열어 그룹 생성/삭제/이름 변경/항목 매핑을 수행한다.
- 모달을 닫으면 즉시 반영한다.
- 그룹 적용 상태에서는 범례에서 각 항목의 소속 그룹(색 인덱스 포함)을 확인할 수 있다.
- 그룹 해제는 `원래대로 보기` 버튼으로 즉시 복귀한다.

### 그래프 유형 전환
- 드롭다운에서 `가로 막대 / 세로 막대 / 원형 그래프 / 누적 막대그래프`를 선택하면 즉시 재렌더링한다.
- 정렬 변경은 현재 선택된 그래프 유형에 동일하게 적용한다.

### 척도 보기 전환
- 척도 문항은 `분포 보기`, `평균 보기`를 전환할 수 있다.
- 일부 raw 척도는 `중간값 제외 보기`를 함께 제공한다.

### 숫자형 보기 전환
- 숫자형 문항은 `히스토그램`, `박스플롯` 전환을 지원한다.
- 그룹 비교 시에는 박스플롯으로 고정한다.

### 순위 범례 제어
- `1순위 / 2순위 / 3순위` 체크박스로 순위별 시리즈 표시를 제어한다.
- `전체 선택 / 전체 해제`를 제공한다.
- 범례 체크 상태는 그래프와 데이터 표에 동시에 반영한다.

### 여러 문항 비교
- 척도형 문항은 대상 문항 여러 개를 한 번에 비교할 수 있다.
- 비교 문항은 현재 선택 문항과 동일한 척도 길이 조건을 만족해야 한다.
- 그룹별로 여러 문항 비교를 볼 때는 분포 보기를 사용할 수 없다.

### 그룹 비교 그래프 동작
- 그룹 비교를 적용해도 기본 차트 레이아웃은 유지한다.
- 잘못 겹치거나 단색으로만 표현되는 비정상 렌더링은 허용하지 않는다.

## 7. 기타 응답 처리 규칙

### 기타 텍스트 컬럼 탐색
- 기타 자유응답 텍스트는 전용 expanded 컬럼에서 읽는다.
- 현재 구현은 아래와 같은 이름 패턴을 순차적으로 허용한다.
  - `__기타_텍스트`
  - `__기타 텍스트`
  - `_기타_텍스트`
  - `_텍스트`
- 위 패턴에 정확히 맞지 않아도, 문항명과 기타 텍스트 의미가 함께 드러나는 경우 느슨한 fallback 탐색을 수행한다.

### 기타 응답 표시
- 기타 자유응답이 존재하면 표에서 `응답 보기` 버튼을 노출할 수 있다.
- 모달에서는 중복 제거 없이 실제 응답 텍스트 목록을 보여준다.
- 기타 응답은 차트 표시보다 상세 확인 보조 기능에 가깝게 다룬다.

## 8. 예외 및 비지원 처리

### 데이터 부족
- 문항은 지원 유형이더라도 실제 집계에 필요한 컬럼이 없으면 결과를 그리지 못할 수 있다.
- 이 경우 비지원 또는 표시 불가 상태의 안내 영역을 보여준다. (컴포넌트 스펙은 **2-7** 참조)

### 순위 문항 컬럼 불완전
- 순위형은 순위별 expanded 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.
- raw 값만 있다고 해서 순위 차트를 강제로 생성하지 않는다.

### 중복 문항 컬럼 불완전
- 중복형은 보기별 expanded 이진 컬럼을 찾지 못하면 정상 집계를 수행하지 않는다.

이 섹션은 현재 화면/코드 동작 기준의 최신 스펙이다.
아래 항목은 위 본문 설명보다 우선 적용한다.

### 1) 그래프 유형 선택 방식
- 기존 버튼형 토글 대신 드롭다운 셀렉트를 사용한다.
- 드롭다운 옵션: `가로 막대 / 세로 막대 / 원형 그래프 / 누적 막대그래프`

### 2) 객관식(단일/중복/순위) 공통 표시 제어
- 그래프는 위 드롭다운 기준으로 즉시 재렌더링한다.
- `기본 순서 / 응답률 높은 순` 정렬 옵션을 지원한다.
- 정렬 변경 시 현재 선택된 그래프 유형에 동일하게 적용한다.

### 3) 세로 막대 표시 규칙
- 세로 막대의 막대 간격/폭은 레이블 가독성을 위해 기존 대비 확장한다.
- 막대 내부 수치 라벨 사용 시:
  - 텍스트 색상은 `White`를 기본으로 한다.
  - 기존 대비 1px 작은 폰트 크기를 사용한다.

### 4) 가로 막대형 그래프 수치 레이블 표시 규칙
- 객관식 단일/중복의 가로 막대, 그룹 비교 가로 막대, 가로 100% 누적 막대, 순위 문항의 가로 누적 막대, 척도 분포 막대처럼 막대 길이로 비율을 표현하는 가로 막대형 그래프에 공통 적용한다.
- 가로 막대형 그래프는 별도의 우측 고정 수치 컬럼을 두지 않는다.
- 비율 수치는 막대와 같은 행의 그래프 영역 안에서만 표시한다.
- 막대 길이와 관계없이 수치는 항상 막대 끝 바깥쪽 오른쪽에 표시한다.
- 수치는 행의 주요 결과값으로 보고 `Label-1-strong`(14px, Semibold 600)을 사용한다.
- 수치는 `neutral-900` 계열을 사용해 대비를 확보한다.

### 5) 데이터 테이블 표시 제어
- 하단 데이터 테이블은 기본 렌더링한다.
- 데이터 테이블의 표시/숨김은 `데이터 테이블 숨기기 / 데이터 테이블 펼치기` 토글 버튼으로 제어한다.
- 별도의 `데이터 표 보기` 체크박스는 사용하지 않는다.
- 원형 그래프 보기에서는 원형 그래프와 데이터 테이블을 같은 행에 배치하고, 동일한 데이터 테이블 토글 버튼으로 표 영역을 접고 펼친다.

### 6) 순위 문항 범례 제어
- `1순위 / 2순위 / 3순위` 체크박스를 통해 순위별 시리즈 표시를 제어한다.
- `전체 선택 / 전체 해제`를 제공한다.
- 범례 체크 상태는 그래프와 데이터 표에 동시에 반영한다.

### 7) 그룹 설정 UX
- 우측 상시 노출 그룹 편집 UI 대신 `그룹 설정` 버튼 기반 모달을 사용한다.
- 모달에서 그룹 생성/삭제/이름 변경/항목 매핑을 수행하고, 닫으면 즉시 반영한다.
- 그룹 적용 상태에서는 우측 범례에서 각 항목의 소속 그룹(색 인덱스 포함)을 확인할 수 있다.

### 8) 그룹 비교 그래프 동작
- 그룹 비교를 적용해도 기본 차트 레이아웃은 유지한다.
- 잘못 겹치거나 단색(옅은 회색)으로만 표현되는 비정상 렌더링은 허용하지 않는다.
- 필요 시 `원래대로 보기`(그룹 해제)로 즉시 복귀 가능해야 한다.
