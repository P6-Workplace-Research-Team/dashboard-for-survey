# PRD — 설문조사 분석 대시보드
최종 갱신: 2026-05-01
작성: purple6studio / (주)퍼시스

---

## 1. 제품 개요

### 무엇을 만드는가
설문조사 데이터를 CSV 파일로 업로드하면, 문항별 차트와 그룹 비교 분석을 즉시 볼 수 있는 웹 대시보드를 자동으로 생성한다.

### 왜 만드는가
설문 결과를 분석하려면 보통 Excel이나 SPSS 같은 별도 도구가 필요하고, 차트를 만들어 보고서에 넣기까지 반복 작업이 많다. 
이 서비스는 데이터 파일만 올리면 분석 화면이 바로 만들어지도록 해서 그 과정을 줄인다.

### 누가 쓰는가
설문 데이터를 분석하고 결과를 보고해야 하는 내부 담당자. 데이터 분석 전문가가 아니어도 쓸 수 있도록 설계한다.

---

## 2. 핵심 개념

이 제품을 이해하려면 세 가지 파일 개념을 알아야 한다.

| 파일 | 역할 | 한 행의 의미 |
|------|------|-------------|
| **문항 코드북** | 설문의 설계도. 대시보드가 데이터를 읽는 기준표 | 문항 1개의 정보 (번호, 카테고리, 문항명, 응답 형식 등) |
| **응답 데이터셋_숫자형** | 설문 결과. 계산·집계·차트 생성에 사용 | 응답자 1명의 응답 (숫자 코드) |
| **응답 데이터셋_텍스트형** | 설문 결과. 화면에 텍스트로 표시할 때 사용 | 응답자 1명의 응답 (텍스트 라벨) |

세 파일은 항상 같은 설문의 것이어야 하며, 문항 구조와 응답자 수가 일치해야 한다.

## 3. 화면 구성

### 3.1 홈 화면 (`home.html`)
대시보드를 새로 만들거나 기존 대시보드를 여는 진입점.

**주요 기능**
- 설문 제목 입력 (최대 50자)
- 3종 CSV 파일 업로드 (드래그&드롭 또는 파일 선택)
- 파일 검증 (형식, 필수 컬럼, 3종 파일 간 정합성)
- 저장된 대시보드 리스트 조회 / 이름 변경 / 삭제

**흐름**
```
제목 입력 → 3종 파일 업로드 → 검증 통과 → 대시보드 화면으로 이동
```

### 3.2 대시보드 화면 (`dashboard.html`)
분석 구조를 정의하고 차트를 보는 메인 화면.

**주요 기능**
- 좌측 문항 패널: 코드북 기반 문항 트리 (카테고리별 아코디언, 검색)
- 필터 바: 응답자 모집단 필터링
- 분석 구조 정의: 문항을 드래그해 두 드롭존에 배치
  - 보고 싶은 문항 (최대 20개)
  - 그룹별 비교 (최대 1개)
- 결과 영역: 문항 유형에 맞는 차트 + 데이터 테이블
- 데이터 교체하기: CSV 파일 교체 및 재검증
---

## 4. 파일 검증 규칙

### 4.1 업로드 시 즉시 실행 (1차)

**문항 코드북**
- 필수 컬럼 5개 존재 여부: `question_no`, `question_label`, `response_type`, `data_column_role`, `value_code_map`

**응답 데이터셋 (숫자형 / 텍스트형 공통)**
- 첫 번째 열이 `survey_year`, 두 번째 열이 `respondent_no`인지 확인
- 첫 번째 데이터 행의 `survey_year` 값이 4자리 연도인지 확인

**대시보드 만들기 클릭 시 (코드북 기반 추가 판별)**
- 코드북의 `value_code_map`이 있는 객관식 단일/중복/순위 raw 문항을 찾아 숫자형/텍스트형 파일이 뒤바뀌었는지 확인 (판별 가능한 문항이 없으면 통과)

### 4.2 대시보드 만들기 버튼 클릭 시 실행 (2차)

3종 파일 간 정합성 검증:
- 코드북 ↔ 숫자형: 문항 이름과 순서 일치 여부
- 코드북 ↔ 텍스트형: 문항 이름과 순서 일치 여부
- 숫자형 ↔ 텍스트형: 헤더 구조, 행 수, 기준 컬럼 값, 응답 위치 구조 일치 여부

## 5. 지원 문항 유형과 시각화

| 문항 유형 | 기본 차트 | 그룹 비교 | 여러 문항 비교 |
|----------|---------|---------|--------------|
| 객관식 단일 | 가로/세로 막대, 원형, 누적 | ✓ | — |
| 객관식 중복 | 가로/세로 막대 | ✓ | — |
| 객관식 순위 | 롤리팝, 누적 막대 | ✓ (1순위 기준) | — |
| 객관식 척도 | 분포 (누적 막대), 평균 마커 | ✓ | ✓ (같은 척도 길이) |
| 주관식 숫자 | 히스토그램, 박스플롯 | ✓ | — |
| 주관식 비율 배분 | 누적 막대 (항목별) | ✓ | — |
| 주관식 문자 | 미지원 | — | — |

## 6. 데이터 저장 방식

> **TBD** — Supabase 기반 서버 저장소로 전환 검토 중. 현재는 브라우저 저장소 사용.

| 저장소 | 저장 내용 |
|--------|---------|
| localStorage | 설문 메타 정보 (제목, 생성일, 파일 참조 등) |
| IndexedDB | 실제 CSV 파일 텍스트 |
| sessionStorage | 화면 간 임시 데이터 (설문 ID, 제목) |

- 동일한 파일은 해시 기반 키로 중복 저장하지 않음
- 삭제 시 다른 설문이 같은 파일을 참조 중이면 파일 데이터는 보존

## 8. 이미지 추출

차트를 고객사 제출용 이미지로 저장할 수 있다.

- 출력 사양: 1920×1080px, PNG, 흰 배경
- 각 결과 섹션마다 "이미지로 저장" 버튼 제공
- 내부 UI 컨트롤(그래프 유형 선택, 정렬 버튼 등)은 이미지에서 제외
- 차트 높이가 1080px를 초과하면 복수 페이지로 분할 저장
