# 인식DB 대시보드 현재 UI/기능 정의서

이 문서는 현재 [home.html](C:\Users\user\Documents\Claude\Projects\인식DB 대시보드_\home.html)과 [dashboard.html](C:\Users\user\Documents\Claude\Projects\인식DB 대시보드_\dashboard.html)에 반영된 실제 화면, 디자인, 기능, 검증 규칙을 사람이 이해하기 쉽게 정리한 문서이다.

목적은 다음과 같다.

- 현재 프로토타입이 어디까지 구현되었는지 빠르게 파악
- 홈 화면과 대시보드 화면의 역할 구분
- 화면별 주요 UI 요소와 동작 정의
- 업로드/데이터 업데이트 시 적용되는 검증 규칙 정리
- 추후 디자인 시스템, 기능 확장, 개발 분리 시 기준 문서로 활용

## 1. 전체 서비스 흐름

현재 프로토타입의 기본 흐름은 아래와 같다.

1. 홈 화면에서 설문 제목을 입력한다.
2. 코드북, 숫자형 응답 데이터, 라벨형 응답 데이터를 업로드한다.
3. 파일 검증과 교차 검증이 모두 통과하면 대시보드 생성이 가능해진다.
4. 대시보드 화면에서 문항을 탐색하고, 분석 대상/기준 문항을 드래그해 배치한다.
5. 상단 필터를 적용해 표본 수와 결과 화면을 확인한다.
6. 현재 상태를 저장하거나, 저장된 대시보드 목록을 다시 열어 불러온다.
7. 필요하면 대시보드 상단의 `데이터 업데이트` 기능으로 원본 파일 일부를 교체한다.

## 3. 홈 화면 정의

### 3.1 화면 목적

홈 화면은 새 설문조사 분석 대시보드를 시작하는 진입 화면이다.

사용자는 이 화면에서 다음을 수행한다.

- 설문 제목 입력
- 코드북 업로드
- 응답 데이터셋_숫자형 업로드
- 응답 데이터셋_라벨형 업로드
- 저장된 대시보드 목록 확인
- 분석 시작

### 3.2 레이아웃 구조

홈 화면은 크게 5개 영역으로 구성된다.

1. 상단 보조 바
2. 메인 제목 영역
3. 제목 입력 섹션
4. 파일 업로드 섹션
5. 가이드 배너 및 저작권

주요 레이아웃 특징:

- 전체 콘텐츠 최대 폭은 `900px`
- 좌측 상단에 `purple6studio` 로고
- 우측 상단에 `저장된 대시보드 리스트` 버튼
- 상단 보조 요소와 메인 제목 사이에 연한 구분선 사용
- 첫 화면에서 모든 핵심 요소가 한눈에 들어오도록 세로 밀도 조정

### 3.3 주요 UI 요소

#### 상단 보조 바

- 좌측: `purple6studio` 로고
- 우측: `저장된 대시보드 리스트` 버튼 + 저장 개수 badge
- 리스트 버튼은 홈/대시보드 화면에서 동일 계열 UI 사용

#### 메인 제목

- 텍스트: `설문조사 분석 대시보드 만들기`
- 보조 요소보다 더 강한 위계로 표현

#### 설문 제목 입력

- 입력 필드 1개
- placeholder 예시 제공
- `maxlength="50"`으로 실제 입력 길이 제한
- 안내 문구: 최대 50자
- 제목이 비어 있으면 에러 상태 표시

#### 파일 업로드 카드 3종

- 문항 코드북
- 응답 데이터셋_숫자형
- 응답 데이터셋_라벨형

공통 동작:

- 드래그앤드롭 또는 파일 선택 가능
- 허용 확장자: `csv`, `xlsx`
- 업로드 성공/실패 상태가 카드 내부에 반영됨
- 업로드 완료 시 파일명 표시

#### 분석 시작 버튼

- 제목과 3개 파일이 모두 유효할 때만 활성화

#### 가이드 배너

- 하단 배너 형태
- 현재는 placeholder 상태
- 클릭 시 실제 이동 대신 안내 alert 동작

#### 저장된 대시보드 리스트 모달

- 저장된 설문 목록 표시
- 저장일 정보 표시
- 클릭 시 해당 설문 열기
- 이름 바꾸기 / 삭제 가능

### 3.4 홈 화면 검증 규칙

홈 화면 업로드는 `개별 파일 검증`과 `파일 간 교차 검증`으로 나뉜다.

#### 1. 개별 파일 검증

##### 코드북 검증

필수 컬럼:

- `question_no`
- `question_label`
- `response_type`
- `data_column_role`

##### 응답 데이터 검증

필수 컬럼:

- `survey_year`
- `respondent_no`

또한 샘플 데이터 기반으로

- 숫자형 데이터인지
- 라벨형 데이터인지

를 자동 판별한다.

#### 2. 파일 간 교차 검증

개별 파일이 각각 정상이어도, 서로 같은 설문 구조를 따르는지 추가로 검사한다.

##### 코드북 vs 응답 데이터셋 구조 일치

비교 기준:

- 코드북의 `question_label` 목록
- 응답 데이터셋의 가로 첫행에서 `survey_year`, `respondent_no`를 제외한 문항 헤더 목록

검사 항목:

- 문항 수가 같은지
- 문항 순서가 같은지
- 문항 이름이 같은지

즉, 코드북의 문항 리스트와 숫자형/라벨형 응답 데이터셋의 문항 리스트가 동일해야 한다.

##### 숫자형 vs 라벨형 응답 데이터셋 구조 일치

검사 항목:

- 가로 첫행 전체 구조가 같은지
- 전체 행 수가 같은지
- 각 행의 `survey_year`, `respondent_no` 위치 값이 같은지
- 각 셀의 값 존재 위치가 같은지

여기서 “값 존재 위치가 같다”는 뜻은,

- 같은 행, 같은 문항 컬럼에서
- 숫자형은 값이 있는데 라벨형은 비어 있거나
- 라벨형은 값이 있는데 숫자형은 비어 있는 경우

를 허용하지 않는다는 의미다.

즉 두 응답 데이터셋은 같은 구조와 같은 응답 위치를 가져야 하고, 값의 표현 형식만 달라야 한다.

### 3.5 저장 구조

설문 정보는 브라우저 `localStorage`의 `p6s.surveys` 키에 저장된다.

포함 정보:

- 설문 id
- 설문 제목
- 업로드 파일 정보
- 생성 시각
- 수정 시각

현재 설문 컨텍스트는 `sessionStorage`에 저장된다.

주요 키:

- `survey.currentId`
- `survey.title`

## 4. 대시보드 화면 정의

### 4.1 화면 목적

대시보드 화면은 업로드한 설문 데이터를 바탕으로 문항을 탐색하고, 분석 대상 문항과 기준 문항을 배치하며, 필터를 적용해 결과를 보는 작업 화면이다.

### 4.2 전체 레이아웃 구조

대시보드 화면은 크게 4개 영역으로 구성된다.

1. 상단 헤더
2. 좌측 문항 리스트 패널
3. 메인 분석 영역
4. 모달 영역

### 4.3 상단 헤더

구성 요소:

- `purple6studio` 로고
- 현재 설문 제목
- 제목 수정 버튼
- `데이터 업데이트` 버튼
- `현재 대시보드 저장하기` 버튼
- `저장된 대시보드 리스트` 버튼
- `새 대시보드 만들기` 버튼

주요 특징:

- 로고는 클릭해도 홈으로 이동하지 않음
- 설문 제목이 로고보다 더 중요하게 보이도록 위계 조정
- 제목 수정은 PNG edit 아이콘 사용
- `현재 대시보드 저장하기`는 주요 CTA
- `데이터 업데이트`는 보조 액션
- `저장된 대시보드 리스트`와 `새 대시보드 만들기`는 관리 액션

### 4.4 좌측 문항 리스트 패널

#### 패널 구조

좌측 패널은 아래 요소로 구성된다.

- 상단 툴바
- 문항 검색창
- 문항 트리
- 선택 상태 바
- 우측 세로 확장 핸들

#### 문항 목록 구성 방식

문항 리스트는 홈 화면에서 업로드한 코드북을 읽어 동적으로 생성된다.

기준 컬럼:

- `category_1`
- `category_2`
- `question_label`
- `question_full`
- `question_no`

렌더링 방식:

- `category_1`: 1차 아코디언
- `category_2`: 2차 아코디언
- `question_label`: 드래그 가능한 문항 카드

노출 규칙:

- 코드북의 `data_column_role = expanded` 행은 사용자용 문항 리스트에 표시하지 않는다.
- `expanded` 행은 중복응답/순위응답 등의 계산 편의를 위한 내부 컬럼으로만 사용한다.
- 따라서 좌측 문항 리스트에는 사용자가 직접 선택해야 하는 문항만 노출한다.

#### 문항 카드 표현

기본 상태:

- 문항 카드에는 `question_label`만 표시
- 연한 회색 배경 카드
- hover/selected 상태 존재

확장 상태:

- `question_full` 값이 있는 문항만 `label | Q. full` 형태로 표시
- `question_full`이 없으면 라벨만 유지

#### 검색 기능

검색 대상:

- `category_1`
- `category_2`
- `question_label`
- `question_full`
- `question_no`

검색 시:

- 관련 아코디언 자동 펼침
- 일치하지 않는 문항/카테고리 숨김
- 결과가 없으면 빈 상태 메시지 표시

#### 다중 선택 및 드래그

문항은 여러 개 선택한 뒤 한 번에 드래그할 수 있다.

현재 동작:

- 클릭으로 선택/해제
- 선택 수가 1개 이상이면 하단 선택 바 표시
- 표시 문구: `n개 선택됨`
- `전체 해제` 버튼으로 한 번에 선택 해제

중요한 원칙:

- 드래그 결과로 전달되는 값은 `question_label`만 사용
- 확장 상태에서 `question_full`을 보고 있어도 드롭존에는 라벨만 들어감

#### 패널 확장 기능

패널 오른쪽의 세로 핸들을 클릭하면 좌측 패널이 확장된다.

현재 확장 방식:

- 메인 영역을 아래로 밀지 않음
- 메인 영역 위를 덮는 오버레이
- 필터 바보다 위 레이어에 표시

### 4.5 메인 분석 영역

메인 영역은 크게 세 부분으로 구성된다.

1. 내부 상단 필터 바
2. 드롭존 영역
3. 결과 영역

#### 필터 바

추가 요소:

- 표본 수 `N`

현재 동작:

- 필터는 코드북과 라벨형 응답 데이터를 기준으로 동적으로 생성된다.
- 각 필터의 선택지는 코드북의 `response_options`를 우선 기준으로 사용한다.
- 컴퓨터가 기본 추천 필터를 먼저 노출하고, 사용자는 `+ 필터 추가`로 다른 필터를 추가할 수 있다.
- `survey_year` 값이 2개 이상 존재하면 `조사 연도` 필터를 항상 가장 왼쪽에 고정 노출한다.
- `조사 연도` 필터는 제거할 수 없다.
- 현재 필터 바에 올라온 모든 필터는 제거할 수 있다.
- 제거한 필터는 `+ 필터 추가` 메뉴에서 다시 추가할 수 있다.
- 활성 필터는 드래그해서 순서를 바꿀 수 있다.
- 단, `조사 연도`처럼 고정 필터는 항상 맨 앞에 유지되며 드래그 대상에서 제외된다.
- 필터 드래그 중에는 삽입 위치를 보여주는 세로 구분선이 버튼 사이에 표시된다.
- 필터 드래그 중에는 다른 필터 버튼의 hover 강조를 노출하지 않는다.
- 각 필터는 다중 선택(중복 체크)이 가능하다.
- 실제 라벨형 응답 데이터 기준으로 조건에 맞는 응답자 수를 계산해 `N` 값을 표시한다.

#### 필터 추천 규칙

기본 추천 대상:

- `survey_year` 값이 여러 개인 경우 `조사 연도` 필터를 최우선 고정 필터로 포함
- `category_1 = 응답자 정보`
- `response_type = 객관식 단일`
- `data_column_role = raw`

위 조건을 만족하는 문항은 기본 필터 후보로 우선 포함한다.

추가 가능 대상:

- `response_type = 객관식 단일`
- `data_column_role = raw`
- 실제 라벨형 응답 데이터에 대응 컬럼이 존재하는 문항
- 보기 값이 지나치게 많지 않은 문항

기본 제외 대상:

- `data_column_role = expanded`
- `data_column_role = derived`
- `response_type = 객관식 중복`
- `response_type = 순위형`
- `response_type = 주관식 숫자`
- `response_type = 주관식 문자`

#### 필터 동작 규칙

- 하나의 필터 안에서는 여러 응답값을 동시에 선택할 수 있다.
- 같은 필터 안의 다중 선택은 OR 조건으로 처리한다.
- 서로 다른 필터끼리는 AND 조건으로 처리한다.

예:

- 성별에서 `남`, `여`를 모두 선택
- 업종에서 `제조/생산` 선택

이면 최종 조건은

- `(성별 = 남 또는 여) AND (업종 = 제조/생산)`

으로 해석한다.

#### 드롭존 영역

드롭존은 두 개다.

- 분석 대상 문항
- 분석 기준 문항

제한 수:

- 대상 문항: 최대 20개
- 기준 문항: 최대 2개

동작:

- 좌측 패널에서 드래그해 drop
- 중복 라벨은 추가되지 않음
- 제한 초과 시 alert로 안내
- chip 형태로 표시
- chip에서 제거 가능

#### 결과 영역

현재 결과 영역은 목업 기반이다.

포함 요소:

- 막대 차트 mock
- 빈도/비율 표

현재 상태:

- 값은 고정 샘플 데이터
- 사용자 선택 문항에 실제 계산까지는 아직 미구현

### 4.6 제목 수정 기능

설문 제목은 수정 가능하다.

동작:

- edit 아이콘 클릭 또는 제목 더블클릭으로 수정 모드 진입
- blur / Enter로 저장
- Escape로 취소
- 최대 50자 제한
- 저장된 설문이면 `localStorage` 제목도 함께 갱신

### 4.7 저장된 대시보드 리스트 모달

기능:

- 저장된 설문 목록 표시
- 저장일 표시
- 클릭 시 현재 화면 위에서 바로 설문 전환
- 이름 바꾸기
- 삭제

중요한 점:

- 목록 버튼 클릭 시 홈 화면으로 이동하지 않음
- 현재 대시보드 위에 모달이 뜨는 구조

### 4.8 현재 대시보드 저장 / 새 대시보드 시작

#### 현재 대시보드 저장하기

- 현재 설문 id가 있을 때만 저장 가능
- 저장 시 `updatedAt` 갱신
- 저장 개수 badge 동기화

#### 새 대시보드 만들기

- 현재 `sessionStorage` 컨텍스트 제거
- 홈 화면으로 이동

### 4.9 데이터 업데이트

이미 저장된 대시보드의 원본 파일(코드북/숫자형/라벨형) 중 일부를 열어둔 상태에서 개별 교체할 수 있다. 문항 라벨 수정이나 데이터 보완처럼 가벼운 변경을 반영하기 위한 기능이다.

#### 진입 방식

- 대시보드 헤더의 `데이터 업데이트` 버튼 클릭
- 모달 형태로 열림

#### 모달 구성

- 설명 문구
- 파일별 행 목록
- 행 순서:
  - 문항 코드북
  - 응답 데이터셋 (숫자형)
  - 응답 데이터셋 (라벨형)
- 각 행에는 현재 파일명과 상태 메시지, `파일 교체` 버튼 표시

#### 파일 교체 동작

1. 사용자가 특정 파일 행의 `파일 교체` 클릭
2. CSV/XLSX 파일 선택
3. 개별 파일 검증 수행
4. 현재 저장된 나머지 파일들과 함께 교차 검증 수행
5. 통과 시 해당 파일만 교체 저장
6. 실패 시 교체 취소 및 상태 메시지 표시

#### 데이터 업데이트 시 적용되는 검증 규칙

데이터 업데이트는 홈 화면과 동일한 검증 규칙을 사용한다.

##### 개별 파일 검증

- 코드북: `question_no`, `question_label`, `response_type`, `data_column_role` 필수
- 응답 데이터: `survey_year`, `respondent_no` 필수
- 숫자형/라벨형 여부 자동 판별

##### 교차 검증

- 코드북의 `question_label` 목록과 응답 데이터셋 헤더 일치 여부 검사
- 숫자형/라벨형 데이터셋의 헤더 구조 일치 여부 검사
- 숫자형/라벨형 데이터셋의 행 수 일치 여부 검사
- 숫자형/라벨형 데이터셋의 `survey_year`, `respondent_no` 값 정렬/위치 일치 여부 검사
- 숫자형/라벨형 데이터셋의 응답값 위치 구조 일치 여부 검사

#### 코드북 교체 시 추가 동작

코드북이 교체되면 좌측 문항 리스트가 새 코드북 기준으로 즉시 다시 그려지고, 드롭존의 칩은 다음 규칙으로 정리된다.

- 새 코드북의 `question_label`과 일치하는 칩은 유지
- 새 코드북에 없는 칩은 자동 제거
- 제거된 칩이 1개 이상이면 alert로 안내
- 검색창 값은 초기화

#### 응답 데이터 교체 시 동작

- 파일 정보만 갱신
- 좌측 문항 리스트와 기존 칩 상태에는 직접 영향 없음

## 8. 요약

현재 프로토타입은 “설문 업로드 → 문항 탐색 → 문항 배치 → 필터 적용 → 저장 → 필요 시 원본 파일 업데이트”의 핵심 사용자 흐름이 화면 수준에서 상당 부분 정리된 상태다.

특히 강점은 다음과 같다.

- 홈 화면과 대시보드 화면의 역할이 명확함
- 코드북 기반 문항 패널 구조가 실제 데이터 흐름과 연결됨
- 저장된 대시보드 목록, 제목 수정, 다중 선택 드래그, 데이터 업데이트처럼 실사용에 중요한 보조 기능이 이미 반영됨
- 업로드/업데이트 시 개별 파일 검증뿐 아니라 파일 간 구조 일치 여부까지 검사함

이 문서는 현재 구현된 실제 상태 기준 문서이며, 이후 UI/기능 변경 시 함께 갱신하는 것을 권장한다.

## 9. 최근 반영 사항

아래 항목은 최근 `dashboard.html` 기준으로 추가 반영된 실제 동작이다.

### 9.1 분석 대상 문항 영역

- `분석 대상 문항` 드롭존 헤더에는 `모두 삭제` 버튼이 존재한다.
- `모두 삭제`를 누르면 현재 분석 대상 칩을 한 번에 제거하고, 대상 문항 개수 카운트도 즉시 `0`으로 갱신한다.
- `분석 기준 문항` 영역에는 동일한 일괄 삭제 버튼이 없고, 현재는 개별 칩 제거만 지원한다.

### 9.2 대시보드 헤더 액션

- 대시보드 상단 우측 액션은 아래 4개를 기본으로 제공한다.
  - `데이터 업데이트`
  - `현재 대시보드 저장하기`
  - `저장된 대시보드 리스트`
  - `새 대시보드 만들기`
- `현재 대시보드 저장하기`는 현재 세션의 대시보드 제목과 `updatedAt`을 기준으로 저장 상태를 갱신한다.
- `저장된 대시보드 리스트`는 현재 화면 위에 모달로 열리며, 저장된 항목 열기/이름 바꾸기/삭제를 지원한다.
- `새 대시보드 만들기`는 홈 화면(`home.html`)로 이동한다.

### 9.3 데이터 업데이트 모달의 현재 동작

- `데이터 업데이트` 버튼을 누르면 현재 저장된 대시보드에 연결된 파일 목록을 보여주는 모달을 연다.
- 모달에서는 아래 3개 파일 단위를 각각 교체할 수 있다.
  - 문항 코드북
  - 응답 데이터셋_숫자형
  - 응답 데이터셋_라벨형
- 파일 교체 후에는 현재 저장된 대시보드의 해당 파일 정보와 `updatedAt`을 갱신한다.
- 코드북 교체 후에는 코드북 인덱스, 좌측 문항 리스트, 필터, 결과 영역을 다시 렌더링한다.
- 응답 데이터 교체 후에는 필터와 결과 영역을 다시 렌더링한다.

### 9.4 결과 영역의 현재 레이아웃 보강

- 그룹 비교 차트가 있는 결과 카드에서는 범례를 차트 우측의 세로 패널로 배치한다.
- 화면 폭이 충분하지 않은 경우에는 범례 패널이 차트 아래로 이동할 수 있다.
- `other_input_expected = Y`인 객관식 단일 문항은, 기타 텍스트 열이 존재할 경우 결과 카드 하단에 `기타 응답` 목록 박스를 함께 표시한다.
- 그룹 비교 꺾은선/점은 실제 렌더된 막대 트랙 중심 좌표를 기준으로 정렬한다.

### 9.5 분석 기준 문항 선택 방식 변경

- `분석 기준 문항`은 더 이상 드래그앤드롭 영역으로 두지 않는다.
- 현재는 우측 카드에서 드롭다운으로 `1개 문항만` 선택하는 방식으로 동작한다.
- 드롭다운의 기본값은 `분석 기준 문항 없음`이다.
- 선택 가능한 문항은 현재 데이터셋에 존재하는 `객관식 단일` + `raw` 문항으로 제한한다.
- 분석 대상 문항은 기존처럼 드래그앤드롭으로 최대 20개까지 추가할 수 있다.

### 9.6 데이터 테이블/보조 텍스트 가독성 조정

- 결과 데이터 테이블의 count 열 제목은 `N` 대신 `빈도수 (명)`으로 표시한다.
- 결과 카드 상단의 질문 원문(`question_full`)은 기존보다 조금 더 큰 크기와 높은 대비로 표시한다.
## Rank Question Rules

The current UI applies the following shared rules to every `객관식 순위` result card.

- Rank percentages for `1순위`, `2순위`, `3순위` and beyond use the full respondent count for that question, not the count of people who filled that specific rank.
- Group comparison uses the same rule with each displayed group's respondent count `n`.
- Responses that are present only in the raw rank answer and absent from rank expanded columns are still rendered in the rank chart and rank data table.
- Those raw-only non-ranked responses are excluded from weighted ranking logic, so `가중 점수` and `종합 순위` are shown as `-`.
- `기타` is shown in the chart and data table but always excluded from `종합 순위`.
- `기타` direct-input text is opened by the table's `응답 보기` button in the shared modal, matching single and multiple choice behavior.
- In group comparison, hiding a group in the legend removes it from both visualization and table output.
- Group-level rank summaries are displayed as a table with groups on rows and rank positions on columns.
- Rank data tables keep a total row, and the total row leaves `가중 점수` and `종합 순위` as `-`.
